Data Guide

Use this schema as the baseline payload for the recap engine. It supports standings, pit history, events, and optional replay frames from FastF1 extraction.

Required Blocks

`race`, `drivers`, and `events` should always be present. `replay` and `pitHistory` can be omitted for quick synthetic playback.

Driver Fields

Each driver entry should include `code`, `team`, `pos`, `gap`, and `tyre`. Best lap and tyre age are optional but recommended.

Replay Frames

For real animation, provide timestamped `cars` coordinates: `{ t: number, cars: { VER: { x, y } } }`.

Events

Use lap-based events with a `type` and `message` for race control and broadcast ticker overlays.

{
  "race": {
    "name": "Austrian Grand Prix",
    "year": 2024,
    "circuit": "Red Bull Ring",
    "laps": 71,
    "weather": "Sunny, 28C"
  },
  "drivers": [
    {
      "pos": 1,
      "code": "VER",
      "team": "Red Bull",
      "gap": "LEADER",
      "bestLap": "1:07.275",
      "tyre": "H"
    }
  ],
  "pitHistory": [
    { "driver": "NOR", "lap": 23, "compound": "H", "duration": "2.4s" }
  ],
  "events": [
    { "lap": 1, "type": "start", "message": "Lights out and away we go!" }
  ],
  "replay": {
    "track": [{ "x": 0.0, "y": 0.2 }],
    "frames": [{ "t": 0, "cars": { "VER": { "x": 0.1, "y": 0.2 } } }]
  }
}
